In-plant Safety Emergency
Warning System

Our research introduces an advanced In-Plant Safety Emergency Warning System that uses IoT, RFID, image processing, and machine learning to detect emergencies in real time, pinpoint exact locations, and coordinate rapid, informed responses. This system not only speeds up rescue efforts but also helps prevent future incidents through predictive analysis—ensuring a safer, smarter workplace.

Project Scope

This project develops a smart emergency warning system using IoT, RFID, and machine learning to detect hazards and motor risk level, locate emergencies accurately, and improve safety in industrial environments.

Literature Survey

The integration of intelligent technologies in industrial safety systems has significantly transformed how emergencies are detected, managed, and prevented. Traditional alarm systems often rely on manual triggers or simple threshold-based alerts, which lack precision and adaptability in dynamic industrial environments.

  • Evolution from basic sensor alarms to intelligent IoT-based emergency systems
  • Adoption of machine learning for predictive maintenance and hazard forecasting
  • Risk classification models enhancing predictive insights for motor failures
  • Image processing enabling automated fire detection and visual hazard analysis

Research Gap

Many existing industrial safety systems are reactive, providing only basic alerts without precise location data, intelligent risk assessment, or coordination support. They often fail to authenticate responders or predict potential hazards such as motor failures, leading to delayed responses and operational downtime. There is a significant gap in integrated systems that combine real-time detection, location accuracy, and predictive intelligence to improve both emergency response and hazard prevention.

Current Limitations

  • • Lack of precise location tracking during emergencies
  • • Inability to predict motor failures or future hazards
  • • Delayed and uncoordinated emergency responses

Our Solution

  • • Real-time alert system with accurate location identification
  • • Machine learning models to classify motor risk levels
  • • Image processing for automated fire detection

Research Problem & Solution

Problem Statement

Current in-plant alarm systems are limited in functionality—they indicate the presence of danger but often lack accurate location data, leading to confusion and delays. Emergency responses are not streamlined, and all responders are alerted simultaneously without role-based coordination. Additionally, fires that surpass the internal team's capabilities remain unmanaged, and there is no built-in intelligence to assess motor risk levels, which leads to unexpected machinery failures.

Proposed Solution

The proposed system integrates IoT sensors, RFID authentication, image processing, and machine learning to build a comprehensive emergency management platform. It provides real-time alerts with accurate location data, enables simplified emergency activation with a single button press, authenticates responders to coordinate deployment, triggers external fire services when needed, and uses predictive models to assess and report motor risk levels.

🧩 Core System Modules

Motor Risk Classification

Predicts motor failure risk using live sensor values. Models are trained using Gaussian Naive Bayes, optimized through Grid Search for accuracy.

  • Features: Voltage, Current, Temp, Vibration
  • Outcome: Normal / Low / Moderate / High

Fire Intensity Analysis

Uses live camera feeds to detect flames using color & shape-based filters. Classifies intensity and automates alert triggering.

  • Technique: Color thresholding, edge detection
  • Output: High / Medium / Low fire severity

Responder Authentication

Ensures only trained personnel can deactivate or handle emergencies via RFID tags and secure logging.

  • Hardware: RFID + ESP32
  • Action: Role-based acknowledgment logging

Emergency Detection & Alerting

Uses sensors, buttons, and camera modules to detect abnormal events, pushing alerts to response units in real-time.

  • Devices: IoT nodes, manual triggers
  • Alerts: Location-based visual/audio alerts

⚙️ Technical Implementation

Backend APIs

Built with FastAPI; handles alerts, predictions, and log retrieval.

Authentication

JWT-secured access control for admin panel and responder validation.

Deployment

Hosted on a local XAMPP server with options to expand via cloud deployment.

📈 Evaluation & Results

Predictive Accuracy

Gaussian Naive Bayes (GaussianNB) was selected as the final model. It demonstrated a good balance between training and testing performance, making it a suitable choice for deployment.

System Responsiveness

Average emergency acknowledgment time reduced by 43% using RFID and real-time alerts.
